### 데이터 표현은 무엇을 의미하는건가?
컴퓨터가 이하는 방식이 아닌 개발자가 이해할 수 있는 표현법을 의미한다.<br/>
간단하게 보면 수십개의 수치가 적혀 있는 테이블과 해당 수치를 활용한 하나의 그래프가 있다면 뭐가 이해하기 쉬울까?<br/>
코드에서도 컴퓨터가 이해할 수 있는 언어를 개발자가 보기 쉽게 만드는 방식이 있는데, 
Constant, Variable 그리고 comment가 있다고 한다.<br/>

## 세부 내용 + 코드 활용
### Constant
*let으로 표현하며, 변할 수 없는 값을 선언할 때 사용한다.*<br/>
후술하겠지만, variable와 constant는 메모리 위에 데이터를 저장할 수 있도록 선언하는 키워드이다.<br/>

### Variable
*var 키워드로 표현하며, 변할 수 있는 값을 선언할 때 사용한다.*<br/>
Lazy var이라고 값을 선언하는 시점을 미루는 키워드도 있는데, 이건 추후 작성해보겠다!
<br/><br/>
둘 다 '변수'를 선언하는데 같은 역할을 하지만,<br/>
immutable (변할 수 없는지), mutable(변할 수 있는지)에 따라 선언을 다르게 해야한다.<br/>
위 차이점이 가장 큰데, 메모리에 차이점이 있는지는 모르겠다... (이건 추가 확인해봐야겠다!)
<br/>
```Swift
var myAge = 10
let myName = "Madman"
// 나이는 매해 지나면 한 살을 더 먹는 반면, 이름은 변하지 않는다.
// 따라서 나이는 var, 이름은 let으로 선언한 부분을 이해하자.
```

물론 이름도 개명을 하거나 하면 변경이 가능한 요소라 볼 수 있지만,<br/>
코드 상에서, 프로그램 내에서 ID라고 생각하면 PW보다 더 많이 바꾸는 경향이 있는가?
<br/> 모든 건 모바일 환경에서, 어플 내에서 우리가 하는 행동을 대표할 데이터 들임을 생각해보자!
